# 🇧🇩 MVP গাইডলাইন এবং স্ট্যাটাস রিপোর্ট

এই ডকুমেন্টটি বর্তমান প্রজেক্টের MVP (Minimum Viable Product) স্ট্যাটাস এবং গাইডলাইন নির্দেশ করে।

> **আপডেট**: ১৫ জানুয়ারি, ২০২৬
> **স্ট্যাটাস**: MVP প্রায় সম্পন্ন (Customer Management UI তৈরি করা হয়েছে)

---

## 📋 MVP চেকলিস্ট

নিচে ক্যাটাগরি অনুযায়ী ফিচারগুলোর বর্তমান অবস্থা দেওয়া হলো:

### ১. অথেন্টিকেশন (Authentication) ✅ সম্পন্ন

- [x] ইমেল/পাসওয়ার্ড লগইন ও রেজিস্ট্রেশন
- [x] গুগল লগইন (Google Social Login)
- [x] পাসওয়ার্ড রিসেট ফ্লো

### ২. প্রোডাক্ট ম্যানেজমেন্ট (Product Management) ✅ সম্পন্ন

- [x] প্রোডাক্ট তৈরি, এডিট এবং ডিলিট (CRUD)
- [x] ভেরিয়েন্ট সাপোর্ট (Size, Color)
- [x] ইমেজ আপলোড এবং গ্যালারি
- [x] ক্যাটাগরি ম্যানেজমেন্ট

### ৩. স্টোরফ্রন্ট (Storefront) ✅ সম্পন্ন

- [x] ডায়নামিক থিম সাপোর্ট
- [x] ল্যান্ডিং পেজ মোড এবং মাল্টি-পেজ স্টোর মোড
- [x] কার্ট এবং চেকআউট পেজ
- [x] মোবাইল রেসপন্সিভ ডিজাইন

### ৪. অর্ডার ম্যানেজমেন্ট (Order Management) ✅ সম্পন্ন

- [x] অর্ডার লিস্ট এবং স্ট্যাটাস ফিল্টার
- [x] অর্ডারের বিস্তারিত বিবরণ
- [x] ম্যানুয়াল অর্ডার তৈরি (Admin)
- [x] কুরিয়ার ইন্টিগ্রেশন (Steadfast/Pathao)

### ৫. কাস্টমার ম্যানেজমেন্ট (Customer Management) ✅ সম্পন্ন (নতুন)

- [x] **কাস্টমার লিস্ট**: নাম, ইমেল বা ফোন দিয়ে সার্চ করার সুবিধা।
- [x] **কাস্টমার প্রোফাইল**: অর্ডারের ইতিহাস, মোট খরচ (Total Spent) এবং ঠিকানা দেখা।
- [x] **নেভিগেশন**: ড্যাশবোর্ডে "Customers" মেনু যোগ করা হয়েছে।

### ৬. পেমেন্ট ও চেকআউট (Payment) ✅ সম্পন্ন

- [x] ক্যাশ অন ডেলিভারি (COD)
- [x] ম্যানুয়াল পেমেন্ট (bKash/Nagad - মার্চেন্ট নম্বর দেখানো)
- [x] চেকআউট প্রসেস

### ৭. সেটিংস ও কনফিগারেশন (Settings) ✅ সম্পন্ন

- [x] স্টোরের নাম, লোগো, এবং কারেন্সি সেটিংস
- [x] শিপিং চার্জ কনফিগারেশন (ঢাকার ভিতরে/বাইরে)
- [x] পেমেন্ট মেথড সেটআপ

### ৮. এনালিটিক্স (Analytics) ✅ সম্পন্ন

- [x] ড্যাশবোর্ড ওভারভিউ (Sales, Orders)
- [x] টপ প্রোডাক্ট এবং কাস্টমার রিপোর্ট

## 🚀 পরবর্তী পদক্ষেপ (Next Steps)

MVP এখন প্রায় সম্পূর্ণ। পরবর্তী ধাপে আমরা যা ফোকাস করব:

১. **Quick Builder যাচাই**: নতুন স্টোর তৈরির ফ্লো ৫ মিনিটে শেষ করা যাচ্ছে কিনা তা টেস্ট করা।
২. **এন্ড-টু-এন্ড টেস্টিং**: পুরো সিস্টেমটি (অর্ডার থেকে ডেলিভারি) একবার চালিয়ে দেখা।
৩. **পলিশিং (Polish)**: ছোটখাটো UI ইম্প্রুভমেন্ট এবং বাগ ফিক্সিং।

## 🛠 ডেভেলপমেন্ট রুলস (Development Rules)

১. **Atomic Commits**: প্রতিটি ফিচারের পর কমিট করতে হবে।
২. **Tailwind CSS**: স্টাইলিংয়ের জন্য Tailwind ব্যবহার করুন।
৩. **Reuse Components**: একই কোড বারবার না লিখে কম্পোনেন্ট তৈরি করুন।
৪. **Lucide Icons**: আইকনের জন্য `lucide-react` ব্যবহার করুন।
